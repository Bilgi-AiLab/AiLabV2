Project

Create a Project

What we call the project actually refers to the collection of documents. You can touch the + icon on the Projects tab to create a project. When creating a project, just enter your project name and add your documents. Each document should be found as a file.

We currently only support .txt and .pdf file formats.

Add New Files To Project

Go to the Projects tab and click on the project you want to add new files, press the Add New Files button.

Remove Project

Go to the Projects tab and click on the project you want to delete and click the delete project button on the page that opens.

Document Similarity

Select Algorithm

Select a project from the Projects tab, press Apply Document Similarity and choose one of the algorithms you can apply:

  1. TFIDF-Cosine Similarity

  2. TFIDF-Euclidean Distance

  3. TFIDF-Manhattan Distance

  4. Word2Vec-Cosine Similarity

  5. Word2Vec-Euclidean Distance

  6. Word2Vec-Manhattan Distance

Apply Algorithm

After selecting one of the Document Similarity algorithms, you should select a document from the project to compare with other files in the same project and click Run button.

Explore Results

After applying the algorithm, they are not lost and saved as a report, you do not need to re-apply them when you want to look at them. You can click History and click View.

It comes with rank of loading files that contain similarities rate table as default, if you want to rank in the table according to their similarity Similarity / Distance Click on the title.

If you want to download the files, click the Download Files button,
you can select all the files or filter them by a certain percentage and download them.

Topic Modeling

Select Algorithm

Topic modeling is a statistical modeling technique used to uncover hidden thematic structures within a set of texts. It's commonly applied to large text corpora to identify the topics that documents contain and to categorize these documents by their topics. This method enables the extraction of meaningful patterns from vast amounts of data, improving the understanding of document content. It's especially favored in areas like digital libraries, information retrieval systems, and analysis of customer feedback.

Select a project from the Projects tab, press Apply Topic Modeling and choose one of the algorithms you can apply:

  1. LDA

  2. LSA

  3. HDP

  4. NMF

  5. Word2Vec Kmeans

  6. DistilBERT

  7. BERTopic

  8. Top2Vec

Apply Algorithm

After selecting one of the Topic Modeling algorithms, enter number of topics to create and click Run button. (except HDP and Top2Vec algorithm, it does not require topic number)

For LDA, LSA, NMF and Word2Vec models, to find optimum number of topic count, you can use Step section. Put start, end and step numbers to run and create coherence value graph for each topic number.

Example:

Start: 2 , End: 10, Step: 1 will calculate for 2, 3, 4, 5, 6, 7, 8, 9 topics and returns their coherence value as graph. Since it is running for multiple times, it can take a while.

Additionally, for BERTopic, model could be automatically applied without entering number of topics.

Explore Results

After applying the algorithm, they are not lost and saved as a report, you do not need to re-apply them when you want to look at them. You can check History and click View.

There is 3 section in Topic Modeling report.

Topic Word Distribution Table

By default:

  • Table is sorted by Topic Number, you can click the table header to sort them.

  • Topic names are from Topic 1 to Topic N, you can change it by clicking Edit Topic Names, after you rename it click Apply button.

  • In words column 10 keywords are showing respect to their importance, you can change it from 1 to 20, to do that click Filter and use slider.

  • Also, the topic Word distributions output could be downloaded as a PNG file by clicking "Download as PNG".

You can preview and download files by clicking Count of Documents. When you click Download files button, it will download all of the documents related with that topic.

Document Topic Distribution

By selecting a document and clicking Add button, you can see topic distribution of that document. If you want to add all documents at once you can click Show All button.

TSNE Graphs

You can visualize document topic distributions by TSNE graphs 2D or 3D. To see the graphs click Load Graph button.

Sentiment Analysis

Select Algorithm

Sentiment analysis is a natural language processing (NLP) technique used to identify the emotional tone within texts. It automatically detects positive, negative, or neutral sentiments in various texts such as comments, social media posts, and product reviews.

Select a project from the Projects tab, press Apply Sentiment Analysis and choose one of the algorithms you can apply:

  1. VADER

  2. Textblob

  3. DistilBERT

  4. RoBERTa

  5. BERTurk

Apply Algorithm

After selecting one of the Sentiment Analysis algorithms, enter the number of topics to create and click the Run button.

Explore Results

After applying the algorithm, they are not lost and saved as a report, you do not need to re-apply them when you want to look at them. You can check History and click View.

There are 4 sections in the Sentiment Analysis report.

Summary

In Summary; the algorithm, polarity value, and document counts are displayed.

Polarity value is a numerical measure in sentiment analysis that determines whether a text is positive (+), negative (-), or neutral (0). Ranging from -1 to 1, this value indicates the intensity of sentiment, with negative values representing negative sentiment, positive values indicating positive sentiment, and values close to zero meaning neutrality.

Bar Chart

A bar chart is visualized in order to see results.

Pie Chart

A pie chart is also visualized to see a different perspective of the results.

Detailed Scores

In each text, the scores of the text are displayed. Also, the scores can be downloaded as an Excel file by clicking the "Download Excel" button.

Summarization

Select Algorithm

Summarization is the process of automatically condensing a large body of text into a shorter version while retaining its key information and meaning. It is a key application of Natural Language Processing (NLP), often used to help users quickly understand lengthy content.

Select a project from the Projects tab, press Apply Summarization and choose one of the algorithms you can apply:

  1. Pegasus

  2. BART

  3. T5

  4. BRIO

  5. DeepSeek

  6. LexRank

Apply Algorithm

After selecting one of the Summarization algorithms, enter the number of topics to create and click the Run button.

Explore Results

After applying the algorithm, they are not lost and saved as a report, you do not need to re-apply them when you want to look at them. You can check History and click View.

There are 3 sections in the Summarization report.

Generated Summary

In the generated summary, the summarized text is displayed.

BERT Score

BERTScore evaluates semantic similarity by using vector representations generated by the BERT model instead of directly comparing words. It focuses not only on word matches but also on the contextual meaning of words. This makes it a more reliable metric for tasks such as summarization, machine translation, and text generation.

ROUGE Scores

ROUGE scores measure the similarity between the generated text and the reference text. ROUGE-1 (unigram), ROUGE-2 (bigram), and ROUGE-L (length-dependent matches) are evaluated using Precision (accuracy), Recall (coverage), and F1-score (balance metric). A high Precision indicates that many words from the reference text appear in the generated text, while a low Recall suggests that the generated text does not fully cover the information in the reference.
